## Testing Environment

**DonkeySwap** - Intentionally vulnerable Coin swap protocol.

Use a [DonkeySwap](https://github.com/Zellic/DonkeySwap) intentionally vulnerable, Coin-swapping protocol for Aptos. It's basically an AMM that uses an Oracle for its pricing, alongside LP coins minted, representing the USD value of liquidity `deposits/withdrawals`. Featuring support for "limit swap" orders. The purpose of this repository is to demonstrate common vulnerabilities seen in **Aptos** protocols, `@see` also: [aptos-core](https://github.com/aptos-labs/aptos-core) layer-1 blockchain protocol.

**Workshop**

There is [Neodyme Labs](https://github.com/neodyme-labs/neodyme-breakpoint-workshop) Solana Security Workshop for security orienteded tactics and hacking techniques for Solana-based applications. The workshop is designed to provide hands-on experience in identifying and exploiting vulnerabilities in Solana smart contracts, as well as understanding the underlying security principles and best practices for secure development on the Solana blockchain.

Alternatively, there is a [Damn Vulnerable DeFI](https://www.damnvulnerabledefi.xyz) project, which is an intentionally vulnerable DeFi protocol designed to help security researchers and developers learn about common vulnerabilities in decentralized finance applications. The project includes a variety of smart contracts that simulate real-world DeFi scenarios, allowing users to practice identifying and exploiting vulnerabilities in a safe and controlled environment. Take a look at [The Auditooor Grindset](https://www.zellic.io/blog/the-auditooor-grindset/) blogpost and read it carefully, as it provides a great insight into the world of smart contract security and the mindset required for effective auditing.
